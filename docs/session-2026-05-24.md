### 3. Bug Fixes

#### Tag item counts showing zero

**Problem:** `GET /api/tags` returned tags without any item count. The `findAllTags` query was a plain `SELECT * FROM tags` with no join.  
**Fix:** Updated `findAllTags` in `relationships.repository.ts` to LEFT JOIN `item_tags` and return `COUNT(it.id)::int AS item_count`, matching the shared `TagWithCount` type.

#### Tag items list empty on native app

**Problem:** `getItemsByTag` in the mobile API client expected a paginated result `{ items: Item[] }` but the server returns a plain `Item[]` array.  
**Fix:** Updated `src/api/tags.ts` return type to `Item[]` and `setItems(res)` directly in the tag screen.

#### Items list 500 error

**Problem:** No `GET /` route existed in the items router. Requests fell through to an unrelated wildcard handler.  
**Fix:** Added `findPaginated` repo function, `listItems` service + controller, and `GET /` route to `items.routes.ts`.

#### Login 403 behind Apache

**Problem:** `express-rate-limit` threw `ERR_ERL_UNEXPECTED_X_FORWARDED_FOR` because Apache sets `X-Forwarded-For` but Express `trust proxy` was false.  
**Fix:** Added `app.set("trust proxy", 1)` before middleware in `app.ts`.

#### Black screen on mobile (ReactSharedInternals.S undefined)

**Problem:** React 18 was hoisted to root `node_modules` alongside React 19 in the monorepo. `react-native` 0.81.5 requires React 19 internals; the wrong version was being resolved.  
**Fix:** Added a custom `resolveRequest` override in `metro.config.js` to force all `require('react')` calls to resolve from the mobile package's local React 19.

#### Empty error message on login failure

**Problem:** Server returns `{ error: { code, message } }` but the client was reading `body.message`.  
**Fix:** Updated error parsing to `body.error?.message ?? body.message ?? res.statusText ?? \`HTTP ${res.status}\``.

#### Expo Go "Sign in" screen (tunnel mode)

**Problem:** `--tunnel` requires an Expo account login prompt before launching.  
**Fix:** Use `--lan` flag instead: `npx expo start --lan --clear`.

#### Route warning "(auth)" not found

**Problem:** `Stack.Screen` name was `"(auth)"` but the actual route is `"(auth)/login"`.  
**Fix:** Updated Stack.Screen names and used `<Redirect>` component instead of `router.replace` in a `useEffect` (which caused a black screen when navigation wasn't ready).

cd /opt/notes-world && sudo docker compose up --build -d app
